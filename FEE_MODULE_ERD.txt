┌─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│                                         DATA FLOW                                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┘

1. SETUP PHASE:
   fee_heads → fee_structures → fee_structure_items
   discount_types → student_discounts
   
2. CUSTOMIZATION PHASE:
   students → student_fee_assignments
   students → student_discounts
   
3. VOUCHER GENERATION:
   fee_structure_items + student_discounts + student_fee_assignments
   → fee_vouchers → fee_voucher_items
   
4. PAYMENT RECORDING:
   fee_payments → fee_payment_allocations → fee_vouchers (update)
   
5. ADVANCE HANDLING:
   fee_payments (excess) → student_fee_wallet_transactions
   student_fee_wallet_transactions → fee_vouchers (adjustment)

1   = One (exactly one)
M   = Many (zero or more)
0..1 = Zero or One (optional)
(FK) = Foreign Key
(PK) = Primary Key
(UNIQUE) = Unique Constraint
(ENUM) = Enumeration Type
(DEPRECATED) = Field to be phased out
